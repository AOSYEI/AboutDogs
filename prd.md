# 犬类知识分享与多元服务网站 - 前端开发提示词

## 项目概述
开发一个面向新手养犬人群的“犬类知识分享与多元服务网站”的前端部分。网站需以“通俗化、场景化、实用性”为核心原则，提供犬类知识科普、服务资源整合与用户互动功能。目标用户是计划养犬的“准新手”和养犬1年内的“新手主人”。前端需实现美观、直观、易用的交互界面，并确保在多种设备上的响应式体验。

## 核心页面与功能模块

### 1. 首页
- **顶部导航栏**: 包含Logo、首页、知识库、服务大厅、社区、个人中心、登录/注册入口。导航需醒目，响应式设计，在移动端变为汉堡菜单。
- **Banner轮播区**: 展示核心功能或推荐内容，如“新手养犬指南”、“热门服务推荐”。
- **新手快速入门专区**: 突出展示“买前必看”和“养犬第一步”两大板块。用卡片式设计，链接到“品种选择”、“渠道鉴别”、“预算清单”和“日常照顾”、“健康护理”、“行为引导”等关键页面。
- **热门知识推荐**: 展示近期热门或编辑推荐的知识文章列表，包含标题、摘要、阅读量、点赞数。
- **精选服务推荐**: 展示推荐的犬类服务（如训练、寄养、医疗），包含服务名称、评分、价格区间、简短介绍。
- **页脚**: 包含关于我们、联系我们、帮助中心、版权信息等。

### 2. 知识库
- **知识分类导航**: 提供侧边栏或顶部标签页分类，分类方式需基于“买前-买后”时间线，避免使用专业术语。分类包括：
    - **买前必读**: 品种选择、渠道鉴别、预算清单、养犬法律责任。
    - **日常照顾**: 喂食指南、如厕训练、清洁美容、用品推荐。
    - **健康护理**: 疫苗接种、常见疾病症状、急救知识、老年犬护理。
    - **行为引导**: 社会化训练、基础指令、不良行为纠正。
- **文章列表页**: 按分类展示文章，支持关键词搜索、按热度/最新排序。每篇文章卡片包含标题、发布日期、摘要、封面图。
- **文章详情页**: 
    - **结构化内容展示**: 文章正文支持图文混排、表格（如“年龄-体重喂食表”）、步骤列表（如“厕所训练分步指南”）、对比表格（如“小病判断症状对比表”）。
    - **用户互动**: 底部设置“点赞”、“收藏”、“评论”功能。
    - **相关推荐**: 文章末尾推荐3篇相关内容。

### 3. 服务大厅
- **服务分类筛选**: 提供服务类型筛选（如“训练师”、“兽医”、“寄养”、“美容”）、按距离排序、按评分排序。
- **服务商列表**: 以卡片形式展示服务提供者。卡片包含：头像/Logo、名称、服务类型、综合评分、简介、服务价格范围、距离。
- **服务商详情页**: 
    - **基本信息**: 详细介绍、资质认证、服务项目列表。
    - **用户评价**: 展示历史用户的评价（文字、评分、图片）。
    - **服务预约入口**: 提供“立即预约”按钮，点击后弹出预约表单（选择服务项目、日期、时间、宠物信息、备注）。

### 4. 用户社区
- **社区动态流**: 展示用户发布的帖子（文字、图片），可按“最新”、“热门”排序。
- **话题标签**: 展示热门话题标签，如#柴犬#、#新手求助#、#训练打卡#。
- **发帖功能**: 用户可发布新帖，支持文字、图片上传，并可选择关联的话题标签。
- **帖子详情页**: 展示帖子完整内容、评论列表，支持评论、点赞。

### 5. 个人中心
- **个人资料**: 用户可编辑头像、昵称、简介、绑定手机/邮箱。
- **我的宠物**: 可添加、编辑、删除宠物档案（名称、品种、生日、性别、照片）。
- **我的收藏**: 展示用户收藏的知识文章。
- **我的预约**: 展示用户的服务预约记录，状态为“待确认”、“进行中”、“已完成”、“已取消”。
- **我的发布**: 展示用户在社区发布的帖子。

## 全局功能
- **用户认证**: 实现登录、注册、找回密码的前端页面和交互，与后端API对接。
- **响应式布局**: 使用Flexbox或Grid布局，确保在PC端、平板、手机端均有良好显示效果。特别是导航菜单、卡片列表、表格等元素需适配不同屏幕。
- **交互反馈**: 所有按钮点击、表单提交需有明确的加载状态、成功或错误提示（如Toast通知）。
- **数据交互**: 前端需通过API（RESTful或GraphQL）与后端进行数据交互。准备好Mock数据或明确API接口数据结构。

## 设计风格与技术栈建议
- **设计风格**: 
    - **整体**: 清新、温暖、亲切，符合宠物主题。
    - **色彩**: 主色调建议使用暖橙色、浅棕色或淡蓝色，搭配柔和的灰色。避免使用过于刺眼的颜色。
    - **字体**: 使用易读的无衬线字体（如Inter, Roboto）。
    - **图片**: 大量使用真实、温暖的养犬家庭照片和可爱的犬类插画。
- **技术栈**: 
    - **框架**: React 或 Vue 3 (使用Composition API)。
    - **UI库**: 可选择不依赖UI库，或使用轻量级库如Tailwind CSS自行构建组件。
    - **状态管理**: 如使用React，考虑Context API或Zustand；如使用Vue，使用Pinia。
    - **路由**: React Router 或 Vue Router。
    - **HTTP客户端**: Axios 或 Fetch API。
    - **构建工具**: Vite。

## 特殊设计要求（依据开题报告）
- **“一看就懂”的内容呈现**:
    - **品种选择页**: 每个品种的卡片需突出“适合人群”、“麻烦事”、“新手友好度（星级）”三个核心信息点。页面需包含真实的家庭养犬照片和“养犬人实话”板块，用生活化场景描述代替专业术语。
    - **日常照顾指南**: 设计“傻瓜式指南”。例如，喂食指南设计成“年龄+体重”的筛选器，动态生成喂食建议表格；如厕训练分解为清晰的步骤卡片；疾病判断提供“症状对比表”，通过点击症状引导至相关建议。
- **新手引导**: 首次访问网站或新注册用户，可弹出简短的新手引导浮层，引导其了解核心功能（知识库、服务大厅、社区）。
